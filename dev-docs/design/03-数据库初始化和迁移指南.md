### 2.3 外键级联策略

⚠️ **重要说明**: 为确保数据一致性和历史数据保留,以下表详细规定了所有外键的级联策略。

#### 2.3.1 完整外键级联策略表

| 关联方向 | 子表 | 外键字段 | 父表 | 级联策略 | 说明 |
|---------|------|---------|------|---------|------|
| **users → strategies** | strategies | user_id | users | CASCADE | 用户删除,策略及其版本、执行记录全部删除 |
| **users → accounts** | accounts | user_id | users | CASCADE | 用户删除,账户及其订单、持仓全部删除 |
| **users → watchlist** | watchlist | user_id | users | CASCADE | 用户删除,自选币种全部删除 |
| **strategies → strategy_versions** | strategy_versions | strategy_id | strategies | CASCADE | 策略删除,版本记录删除 |
| **strategies → strategy_executions** | strategy_executions | strategy_id | strategies | **SET NULL** | 策略删除,执行记录保留但strategy_id为NULL |
| **strategies → backtest_results** | backtest_results | strategy_id | strategies | **SET NULL** | 策略删除,回测结果保留但strategy_id为NULL |
| **strategy_executions → orders** | orders | execution_id | strategy_executions | CASCADE | 执行记录删除,订单删除 |
| **strategy_executions → positions** | positions | execution_id | strategy_executions | CASCADE | 执行记录删除,持仓删除 |
| **strategy_executions → strategy_logs** | strategy_logs | execution_id | strategy_executions | CASCADE | 执行记录删除,日志删除 |
| **strategy_executions → backtest_results** | backtest_results | execution_id | strategy_executions | CASCADE | 执行记录删除,回测结果删除 |
| **strategy_executions → alerts** | alerts | execution_id | strategy_executions | CASCADE | 执行记录删除,告警删除 |
| **accounts → orders** | orders | account_id | accounts | **SET NULL** | 账户删除,订单保留但account_id为NULL |
| **accounts → positions** | positions | account_id | accounts | **SET NULL** | 账户删除,持仓保留但account_id为NULL |
| **accounts → strategy_executions** | strategy_executions | account_id | accounts | **SET NULL** | 账户删除,执行记录保留但account_id为NULL |

#### 2.3.2 级联策略设计原则

**CASCADE (级联删除) - 用于以下场景:**
1. **用户数据**: 用户删除时,其所有相关数据一并删除 (strategies, accounts, watchlist)
2. **策略版本**: 策略删除时,版本记录无独立价值,一并删除
3. **执行记录的子表**: 执行记录删除时,其所有子表数据一并删除 (orders, positions, logs, alerts)
4. **理由**: 这些数据在没有父记录时无独立价值,保留会造成数据垃圾

**SET NULL (置空) - 用于以下场景:**
1. **strategies → executions**: 策略删除后,执行记录应保留用于历史分析和审计
2. **accounts → orders/positions/executions**: 账户删除后,订单和持仓应保留用于交易历史和审计
3. **strategies → backtest_results**: 策略删除后,回测结果应保留用于性能分析对比
4. **理由**: 这些数据具有独立价值,即使父记录被删除也应保留

#### 2.3.3 设计理由

1. **避免数据不一致**:
   - 如果strategies → executions使用CASCADE,会导致历史执行记录丢失
   - 如果accounts → orders使用SET NULL但executions → orders使用CASCADE,会导致孤儿数据
   - 统一的级联策略可以避免复杂的数据不一致问题

2. **保留历史数据**:
   - **策略执行记录**: 即使策略被删除,执行记录、订单、持仓对审计和历史分析仍重要
   - **订单和持仓**: 即使账户被删除,交易历史对问题排查和数据分析仍重要
   - **回测结果**: 即使策略被删除,历史回测结果对性能对比仍重要

3. **便于审计和回溯**:
   - 完整的执行记录可以追溯策略的历史表现
   - 完整的订单记录可以追溯交易历史
   - 完整的回测结果可以对比不同策略的性能

4. **符合数据治理最佳实践**:
   - 重要的业务数据不应因父实体删除而丢失
   - 历史数据应长期保留用于分析和合规

#### 2.3.4 删除流程示例

**场景1: 删除策略**
```python
async def delete_strategy(strategy_id: UUID):
    """删除策略"""

    # 级联删除的表:
    # - strategy_versions (CASCADE)
    #
    # SET NULL的表:
    # - strategy_executions (strategy_id → NULL)
    # - backtest_results (strategy_id → NULL)

    # 执行删除操作
    await db.delete(Strategy).where(Strategy.id == strategy_id)

    # 执行后:
    # - strategy_versions: 全部删除 ✅
    # - strategy_executions: 保留,但strategy_id为NULL ✅
    #   - orders: 保留 ✅
    #   - positions: 保留 ✅
    #   - logs: 保留 ✅
    #   - backtest_results: strategy_id为NULL ✅
```

**场景2: 删除账户**
```python
async def delete_account(account_id: UUID):
    """删除交易所账户"""

    # SET NULL的表:
    # - strategy_executions (account_id → NULL)
    # - orders (account_id → NULL)
    # - positions (account_id → NULL)

    # 执行删除操作
    await db.delete(Account).where(Account.id == account_id)

    # 执行后:
    # - strategy_executions: 保留,但account_id为NULL ✅
    # - orders: 保留,但account_id为NULL ✅
    # - positions: 保留,但account_id为NULL ✅
```

**场景3: 删除执行记录**
```python
async def delete_execution(execution_id: UUID):
    """删除策略执行记录"""

    # - strategy_logs (全部删除)
    # - backtest_results (删除,因为execution_id是唯一键)
```

#### 2.3.6 查询历史数据的建议

当strategy_id或account_id为NULL时,可以通过其他方式查询:

```python
# 查询所有历史执行记录（包括已删除策略的记录）
def get_all_executions():
    """获取所有执行记录"""
    return db.query(StrategyExecution).all()

# 查询特定账户的历史订单（包括已删除账户的订单）
def get_historical_orders_by_symbol(symbol: str):
    """按交易对查询历史订单"""
    return db.query(Order).where(
        Order.symbol == symbol,
        Order.status == "filled"
    ).all()

# 查询策略执行记录（即使策略已删除）
def get_execution_with_strategy(execution_id: UUID):
    """获取执行记录（包含策略信息，如果存在）"""
    execution = db.query(StrategyExecution).where(
        StrategyExecution.id == execution_id
    ).first()

    if execution and execution.strategy_id:
        strategy = db.query(Strategy).where(
            Strategy.id == execution.strategy_id
        ).first()
    else:
        strategy = None

    return execution, strategy
```

## 3. 数据库Schema定义 (SQLAlchemy)

### 3.1 基础模型定义

```python
# models/base.py

from datetime import datetime
from uuid import UUID, uuid4
from sqlalchemy import DateTime, func
from sqlalchemy.dialects.postgresql import UUID as PG_UUID
from sqlalchemy.orm import DeclarativeBase, Mapped, mapped_column


class Base(DeclarativeBase):
    """所有模型的基类"""

    pass


class TimestampMixin:
    """时间戳混入类"""

    created_at: Mapped[datetime] = mapped_column(
        DateTime(timezone=True),
        server_default=func.now(),
        nullable=False
    )
    updated_at: Mapped[datetime] = mapped_column(
        DateTime(timezone=True),
        server_default=func.now(),
        onupdate=func.now(),
        nullable=False
    )


class UUIDMixin:
    """UUID主键混入类"""

    id: Mapped[UUID] = mapped_column(
        PG_UUID(as_uuid=True),
        primary_key=True,
        default=uuid4,
        nullable=False
    )
```

```python
        compare_type=True,
        compare_server_default=True,

    """执行迁移"""

    context.configure(
        connection=connection,
        target_metadata=target_metadata,
        compare_type=True,
        compare_server_default=True,
    )

    with context.begin_transaction():
        context.run_migrations()
```

```sql
-- 创建连续聚合,计算1小时K线
CREATE MATERIALIZED VIEW klines_1h
WITH (timescaledb.continuous) AS
SELECT
    time_bucket('1 hour', open_time) AS bucket,
    exchange,
    symbol,
    interval,
    first(open_price, open_time) AS open_price,
    MAX(high_price) AS high_price,
    MIN(low_price) AS low_price,
    last(close_price, open_time) AS close_price,
    SUM(volume) AS volume,
    SUM(quote_volume) AS quote_volume
FROM klines
GROUP BY bucket, exchange, symbol, interval
WITH DATA;
```

```sql
ON klines(symbol, interval, open_time DESC);

-- 创建部分索引 (只索引特定条件的数据)
CREATE INDEX idx_orders_not_filled
ON orders(created_at)
WHERE status IN ('new', 'partial_filled');
```

### 10.2 分析查询计划
