## 10. 数据库性能优化

### 10.1 创建索引

```sql
-- 创建复合索引 (经常一起查询的字段)
CREATE INDEX idx_orders_execution_status_created
ON orders(execution_id, status, created_at DESC);

CREATE INDEX idx_executions_strategy_status
ON strategy_executions(strategy_id, status);

CREATE INDEX idx_klines_symbol_interval_time
ON klines(symbol, interval, open_time DESC);

-- 创建部分索引 (只索引特定条件的数据)
CREATE INDEX idx_orders_not_filled
ON orders(created_at)
WHERE status IN ('new', 'partial_filled');
```

### 10.2 分析查询计划

```sql
-- 分析查询性能
EXPLAIN ANALYZE
SELECT * FROM orders
WHERE execution_id = 'uuid' AND status = 'filled'
ORDER BY created_at DESC
LIMIT 20;

-- 查看表大小
SELECT
    schemaname,
    tablename,
    pg_size_pretty(pg_total_relation_size(schemaname||'.'||tablename)) AS size
FROM pg_tables
WHERE schemaname = 'public'
ORDER BY pg_total_relation_size(schemaname||'.'||tablename) DESC;

-- 查看索引使用情况
SELECT
    schemaname,
    tablename,
    indexname,
    idx_tup_read,
    idx_tup_fetch
FROM pg_stat_user_indexes
ORDER BY idx_tup_read DESC;
```
