# 量化交易系统 - 开发快速开始指南

---

## 🚀 快速开始

本指南帮助开发团队在10分钟内搭建开发环境并运行系统。

---

## 📋 前置要求

| 要求 | 版本 | 说明 |
|------|------|------|
| **操作系统** | Linux/MacOS/Windows | Linux/MacOS推荐,Windows需WSL2 |
| **Python** | 3.11+ | 主要开发语言 |
| **Docker** | 20.10+ | 容器化部署 |
| **Docker Compose** | 2.0+ | 编排多个容器 |
| **Git** | 2.30+ | 版本控制 |
| **PostgreSQL Client** | 15+ (可选) | 本地连接数据库 |
| **Redis Client** | 7+ (可选) | 本地连接Redis |

### 生成随机密钥

```bash
# 生成SECRET_KEY
python -c "import secrets; print(secrets.token_urlsafe(32))"
```

```bash
# 查看容器日志
docker compose logs postgres

# 重启容器
docker compose restart postgres redis

# 清理并重新启动
docker compose down -v
```


```bash
# 升级pip
pip install --upgrade pip

# 清理缓存
pip cache purge

# 重新安装
pip install -e ".[dev]"
```

```bash
# 查看迁移历史
alembic history

# 查看当前版本
alembic current

# 重置数据库 (慎用!)
alembic downgrade base
alembic upgrade head
```

## 💾 数据备份和恢复

### 自动化备份脚本

创建自动化备份脚本 `scripts/backup_db.sh`:

```bash
#!/bin/bash
# scripts/backup_db.sh
# 数据库自动化备份脚本

set -e

# 配置
BACKUP_DIR="/var/backups/squant"
DATE=$(date +%Y%m%d_%H%M%S)
DB_NAME="squant"
DB_USER="squant"
DB_HOST="${DB_HOST:-localhost}"

# 创建备份目录
mkdir -p "$BACKUP_DIR"

echo "[$(date)] 开始备份数据库..."

# 备份数据库
pg_dump -h "$DB_HOST" -U "$DB_USER" -d "$DB_NAME" -F c \
    -f "$BACKUP_DIR/squant_backup_$DATE.dump"

# 压缩备份
gzip "$BACKUP_DIR/squant_backup_$DATE.dump"

# 保留最近7天的备份
find "$BACKUP_DIR" -name "squant_backup_*.dump.gz" -mtime +7 -delete

echo "[$(date)] 备份完成: squant_backup_$DATE.dump.gz"
echo "[$(date)] 保留最近7天的备份"

# 可选: 上传到云存储 (如AWS S3)
# aws s3 cp "$BACKUP_DIR/squant_backup_$DATE.dump.gz" s3://your-bucket/backups/
```

**使用方法:**

```bash
# 1. 赋予执行权限
chmod +x scripts/backup_db.sh

# 2. 手动执行备份
./scripts/backup_db.sh

# 3. 添加到crontab (每天凌晨2点备份)
crontab -e
# 添加以下行:
0 2 * * * /path/to/squant/scripts/backup_db.sh >> /var/log/squant_backup.log 2>&1
```

### Docker Compose自动备份

在 `docker-compose.yml` 中添加备份服务:

```yaml
services:
  # ... 其他服务 ...

  backup:
    image: postgres:15-alpine
    volumes:
      - ./scripts:/scripts
      - ./backups:/var/backups/squant
    environment:
      - DB_HOST=postgres
      - DB_NAME=squant
      - DB_USER=squant
      - PGPASSWORD=${POSTGRES_PASSWORD}
    command: >
      sh -c "
        apk add --no-cache dcron
        echo '0 2 * * * /scripts/backup_db.sh' | crontab -
        crond -f
      "
    depends_on:
      - postgres
    restart: unless-stopped
```

**启动备份服务:**

```bash
docker compose up -d backup

# 查看备份日志
docker compose logs -f backup

# 查看备份文件
ls -lh ./backups/
```

### 数据库恢复

**方式1: 从备份文件恢复**

```bash
# 解压备份文件
gunzip ./backups/squant_backup_20240124_020000.dump.gz

# 恢复数据库
pg_restore -h localhost -U squant -d squant \
    ./backups/squant_backup_20240124_020000.dump

# 或者恢复到新数据库
createdb -U squant -h localhost squant_restored
pg_restore -h localhost -U squant -d squant_restored \
    ./backups/squant_backup_20240124_020000.dump
```

**方式2: 使用Docker恢复**

```bash
# 恢复到容器中的数据库
docker compose exec -T postgres pg_restore -U squant -d squant \
    < ./backups/squant_backup_20240124_020000.dump
```

### 备份策略建议

| 策略 | 频率 | 保留期 | 位置 |
|------|------|--------|------|
| **本地备份** | 每天 | 7天 | 服务器磁盘 |
| **远程备份** | 每天 | 30天 | 云存储(S3/OSS) |
| **全量备份** | 每周 | 4周 | 云存储 |
| **异地备份** | 每月 | 永久 | 异地数据中心 |

**备份检查:**

```bash
# 检查备份文件完整性
pg_restore -l ./backups/squant_backup_20240124_020000.dump.gz

# 检查备份大小
du -sh ./backups/*

# 检查备份时间
ls -lt ./backups/ | head -10
```
