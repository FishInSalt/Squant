## 1. 系统架构概述

### 1.1 设计原则

基于需求分析和非功能需求,本系统遵循以下设计原则:

| 原则 | 说明 | 应用场景 |
|------|------|----------|
| **单一职责原则 (SRP)** | 每个模块专注于单一业务领域 | 行情模块、策略模块、交易模块分离 |
| **隔离性原则** | 策略运行在独立进程中,互不影响 | 多策略并发运行 (FR-06) |
| **可扩展性原则** | 架构支持横向扩展和市场扩展 | 支持后续扩展港股、美股 |
| **安全性原则** | API Key加密,最小权限原则 | 交易所账户配置 (FR-09) |
| **可观测性原则** | 日志、指标、分布式追踪 | 策略运行监控 (FR-08) |

### 1.2 架构选型与演进策略

⚠️ **重要说明**: 本项目为**个人投资者的个人项目**,资源有限,无大规模团队支持。因此采用**渐进式架构演进策略**,从简化架构开始,根据实际需求逐步演进。

#### 1.2.1 MVP阶段架构 (推荐) ✅

| 架构类型 | 选择 | 理由 |
|----------|------|------|
| **整体架构** | 单体应用 (Monolith) + Docker策略运行 | 满足核心功能,降低维护成本 |
| **部署架构** | Docker Compose | 一键部署,适合本地和云服务器 |
| **通信模式** | REST API + WebSocket | 简单直接,满足前后端交互和实时推送 |
| **内部通信** | 模块间函数调用 (无需RPC) | 简化开发,降低复杂度 |

#### 1.2.2 成熟阶段架构 (按需演进)

| 架构类型 | 选择 | 演进条件 |
|----------|------|----------|
| **整体架构** | 微服务架构 | 策略数量>50 或 需要多团队协作 |
| **API Gateway** | Kong/Traefik | 需要统一认证、限流、路由 |
| **服务间通信** | gRPC | 服务间调用频繁,需要高性能 |
| **消息队列** | RabbitMQ/Kafka | 异步事件量增大,需要可靠消息传递 |
| **编排平台** | Kubernetes | 策略容器数量>100,需要容器编排 |

## 2. MVP单体应用架构设计

### 2.1 模块边界划分 (MVP阶段)

⚠️ **重要说明**: 本节描述的是**单体应用内部的模块划分**,不是微服务边界。

基于领域驱动设计 (DDD),单体应用划分为以下模块:

| 模块名 | 职责 | 优先级 | 说明 |
|--------|------|--------|------|
| **market-module** | 行情数据管理、热门币种、自选币种 | P0 | 负责行情数据的获取、缓存、推送 |
| **strategy-module** | 策略库管理、策略校验、策略版本控制 | P0 | 负责策略文件的上传、校验、存储 |
| **trading-module** | 交易所账户管理、订单管理、持仓管理 | P0 | 负责交易所API集成、订单生命周期管理 |
| **runtime-module** | 策略生命周期管理、运行模式控制、进程隔离 | P0 | 负责策略的启动/停止、Docker容器管理 |
| **monitoring-module** | 运行状态监控、日志聚合、告警通知 | P1 | 负责详细的监控数据收集和展示 |

**模块间通信方式:**

| 场景 | 通信方式 | 说明 |
|------|---------|------|
| 模块A调用模块B | 函数调用 (直接导入) | 同一进程内,直接函数调用,无需RPC |
| 模块间数据共享 | 数据库/Redis | 通过共享存储实现数据传递 |
| 异步事件通知 | Redis Streams | 异步解耦,如订单状态变更通知 |

**与微服务的区别:**

| 方面 | MVP模块 | 成熟微服务 |
|------|---------|-----------|
| **进程数** | 1个进程 | 多个独立进程 |
| **通信方式** | 函数调用 | gRPC/REST |
| **部署** | 单体部署 | 独立部署 |
| **独立性** | 依赖共享代码库 | 完全独立 |
| **适用场景** | 个人项目,策略<50 | 团队协作,策略>50 |

**策略校验规则:**

⚠️ **安全是策略运行的核心要求**,必须实现多层安全防护机制:

#### 1. 语法检查 (AST)
使用 Python `ast` 模块检查语法错误:
```python
import ast

def validate_syntax(code: str) -> tuple[bool, str]:
    """验证策略代码语法"""
    try:
        ast.parse(code)
        return True, "语法检查通过"
    except SyntaxError as e:
        return False, f"语法错误: {e}"
```

#### 2. 接口完整性检查
检查是否实现了必需的接口 (`on_bar`):
```python
def validate_required_methods(code: str) -> tuple[bool, str]:
    """验证必需方法是否实现"""
    tree = ast.parse(code)

    # 查找所有类定义
    classes = [node for node in ast.walk(tree) if isinstance(node, ast.ClassDef)]

    for cls in classes:
        methods = {node.name for node in cls.body if isinstance(node, ast.FunctionDef)}

        required_methods = {'on_init', 'on_bar', 'on_stop'}

        missing = required_methods - methods
        if missing:
            return False, f"类 {cls.name} 缺少必需方法: {missing}"

    return True, "接口检查通过"
```

#### 3. 安全性检查 (AST黑名单)
使用AST遍历检测危险代码模式:
```python
import ast

# 禁止的模块黑名单
FORBIDDEN_MODULES = {
    'os', 'subprocess', 'shutil', 'sys', 'requests', 'socket',
    'http', 'urllib', 'ftplib', 'telnetlib', 'smtplib'
}

# 禁止的函数
FORBIDDEN_FUNCTIONS = {
    'eval', 'exec', 'compile', '__import__', 'open', 'file'
}

class SecurityVisitor(ast.NodeVisitor):
    """AST安全检查访问器"""

    def __init__(self):
        self.violations = []

    def visit_Import(self, node):
        for alias in node.names:
            if alias.name in FORBIDDEN_MODULES:
                self.violations.append(f"禁止导入模块: {alias.name}")
        self.generic_visit(node)

    def visit_ImportFrom(self, node):
        if node.module in FORBIDDEN_MODULES:
            self.violations.append(f"禁止从模块导入: {node.module}")
        self.generic_visit(node)

    def visit_Call(self, node):
        # 检查危险的函数调用
        if isinstance(node.func, ast.Name):
            if node.func.id in FORBIDDEN_FUNCTIONS:
                self.violations.append(f"禁止调用函数: {node.func.id}")
        self.generic_visit(node)

def validate_security(code: str) -> tuple[bool, str]:
    """验证代码安全性"""
    tree = ast.parse(code)
    visitor = SecurityVisitor()
    visitor.visit(tree)

    if visitor.violations:
        return False, "; ".join(visitor.violations)

    return True, "安全检查通过"
```

```python
# 单次回调超时时间（毫秒）
CALLBACK_TIMEOUT_MS = 100

def timeout_check(timeout_ms: int):
    """回调超时装饰器"""
    def decorator(func):
        @wraps(func)
        async def wrapper(*args, **kwargs):
            try:
                return await asyncio.wait_for(
                    func(*args, **kwargs),
                    timeout=timeout_ms / 1000.0
                )
            except asyncio.TimeoutError:
                raise StrategyTimeoutError(
                    f"{func.__name__} 回调超时 (超过 {timeout_ms}ms)"
                )
        return wrapper
    return decorator
```

**错误重试机制:**

交易所API可能因网络问题临时失败,需要实现自动重试。推荐使用`tenacity`库:

```python
from tenacity import (
    retry,
    stop_after_attempt,
    wait_exponential,
    retry_if_exception_type,
    before_sleep_log
)
import logging

logger = logging.getLogger(__name__)


# 可重试的异常类型
RETRYABLE_EXCEPTIONS = (
    aiohttp.ClientError,  # 网络错误
    asyncio.TimeoutError,  # 超时
    ExchangeAPIError,     # 交易所API错误(临时性)
)


class ExchangeAdapter(ABC):
    """交易所适配器抽象基类(带重试)"""

    @retry(
        stop=stop_after_attempt(3),  # 最多重试3次
        wait=wait_exponential(multiplier=1, min=1, max=10),  # 指数退避: 1s, 2s, 4s...
        retry=retry_if_exception_type(RETRYABLE_EXCEPTIONS),
        reraise=True,  # 重试失败后重新抛出异常
        before_sleep=before_sleep_log(logger, logging.WARNING)
    )
    @abstractmethod
    async def place_order(self, request: PlaceOrderRequest) -> Order:
        """下单 (自动重试3次)"""
        pass

    @retry(
        stop=stop_after_attempt(3),
        wait=wait_exponential(multiplier=1, min=1, max=10),
        retry=retry_if_exception_type(RETRYABLE_EXCEPTIONS),
        reraise=True,
        before_sleep=before_sleep_log(logger, logging.WARNING)
    )
    @abstractmethod
    async def cancel_order(self, order_id: str) -> bool:
        """撤单 (自动重试3次)"""
        pass

    @retry(
        stop=stop_after_attempt(2),  # 查询操作重试2次即可
        wait=wait_exponential(multiplier=1, min=0.5, max=5),
        retry=retry_if_exception_type(RETRYABLE_EXCEPTIONS),
        reraise=True,
        before_sleep=before_sleep_log(logger, logging.INFO)
    )
    @abstractmethod
    async def query_balance(self) -> Dict[str, float]:
        """查询余额 (自动重试2次)"""
        pass

    @retry(
        stop=stop_after_attempt(2),
        wait=wait_exponential(multiplier=1, min=0.5, max=5),
        retry=retry_if_exception_type(RETRYABLE_EXCEPTIONS),
        reraise=True,
        before_sleep=before_sleep_log(logger, logging.INFO)
    )
    @abstractmethod
    async def query_positions(self) -> List[Position]:
        """查询持仓 (自动重试2次)"""
        pass


class BinanceAdapter(ExchangeAdapter):
    """Binance交易所适配器(带重试)"""

    def __init__(self, api_key: str, api_secret: str):
        self.api_key = api_key
        self.api_secret = api_secret
        self.session = aiohttp.ClientSession()

    async def place_order(self, request: PlaceOrderRequest) -> Order:
        """下单 (自动重试3次)"""

        url = f"https://api.binance.com/api/v3/order"
        headers = {"X-MBX-APIKEY": self.api_key}

        # 构建请求参数
        params = {
            "symbol": request.symbol,
            "side": request.side,
            "type": request.order_type,
            "quantity": str(request.quantity),
        }

        # 签名
        params["signature"] = self._sign(params)

        try:
            async with self.session.post(url, headers=headers, params=params) as response:
                data = await response.json()

                if response.status != 200:
                    raise ExchangeAPIError(f"Binance API错误: {data}")

                return Order(
                    id=data["orderId"],
                    symbol=request.symbol,
                    side=request.side,
                    order_type=request.order_type,
                    quantity=request.quantity,
                    status=data["status"]
                )

        except Exception as e:
            logger.error(f"下单失败: {e}")
            raise  # tenacity会自动重试

    async def close(self):
        """关闭会话"""
        await self.session.close()
```

| 下单 | 3次 | 指数退避(1s, 2s, 4s) | 关键操作,需要较高的可靠性 |
| 撤单 | 3次 | 指数退避(1s, 2s, 4s) | 关键操作,需要较高的可靠性 |
| 查询余额 | 2次 | 指数退避(0.5s, 1s) | 查询操作,失败影响较小 |
| 查询持仓 | 2次 | 指数退避(0.5s, 1s) | 查询操作,失败影响较小 |

```python
        1. 获取策略配置 (stop_config)
        2. 根据on_stop_action处理订单和持仓

        处理停止时的订单和持仓

        Args:
            execution_id: 策略执行ID
            config: 停止配置

        Returns:
            处理结果统计 (撤销订单数、平仓数)
        """
        action = config.get("on_stop_action", "cancel_orders")

        if action == "cancel_orders":
            # 撤销所有未成交订单
            return await self.cancel_pending_orders(execution_id)
        elif action == "close_positions":

            await self.close_all_positions(execution_id)
            return await self.cancel_pending_orders(execution_id)
        elif action == "do_nothing":
            # 仅停止策略,不操作订单和持仓
            return {"orders_canceled": 0, "positions_closed": 0}
        else:

                # 实盘订单: 调用交易所API撤单
                await trading_module.cancel_order(order.id)
            order.status = "canceled"
            canceled_count += 1

        await db.commit()

        """平掉所有持仓"""
        positions = await db.query(Position).where(
            Position.execution_id == execution_id,
            Position.quantity != 0
        ).all()

        closed_count = 0
        for position in positions:
            if not position.is_simulation:
                # 实盘持仓: 下反向平仓单
                side = "sell" if position.side == "long" else "buy"
                await trading_module.place_order(
                    symbol=position.symbol,
                    side=side,
                    order_type="market",
                    quantity=position.quantity
                )
            position.quantity = "0"
            closed_count += 1

        await db.commit()
        return {"positions_closed": closed_count}
```

| `do_nothing` | 仅停止策略,不操作订单和持仓 | 需要人工干预或手动管理 | 需要人工监控,风险较高 |

```python
"""
策略模板文件
用户基于此模板实现自定义策略
"""

from abc import ABC, abstractmethod
from dataclasses import dataclass
from typing import List, Dict, Optional
from datetime import datetime
from decimal import Decimal
import logging

# 策略SDK (由系统提供)
from squant.strategy_sdk import (
    Context,
    Bar,
    Tick,
    Order,
    OrderType,
    OrderSide,
    OrderStatus,
    Position,
)

logger = logging.getLogger(__name__)


@dataclass
class StrategyConfig:
    """策略配置参数"""
    # 在策略启动时传入
    symbols: List[str]  # 交易对列表,如 ["BTCUSDT", "ETHUSDT"]
    account_id: str  # 交易所账户ID
    run_mode: str  # 'backtest', 'paper', 'live'

    # 策略自定义参数
    param1: float = 0.0
    param2: int = 0
    # ... 更多参数


class BaseStrategy(ABC):
    """
    策略基类
    所有自定义策略必须继承此类并实现抽象方法
    """

    def __init__(self, config: StrategyConfig, context: Context):
        """
        策略初始化

        Args:
            config: 策略配置参数
            context: 策略上下文 (提供行情、交易、日志等API)
        """
        self.config = config
        self.ctx = context
        logger.info(f"策略初始化: {self.__class__.__name__}, 配置: {config}")

    @abstractmethod
    async def on_init(self):
        """
        策略初始化回调
        在策略启动时调用一次,用于初始化策略状态
        """
        pass

    @abstractmethod
    async def on_bar(self, bar: Bar):
        """
        K线数据回调

        Args:
            bar: K线数据
        """
        pass

    async def on_tick(self, tick: Tick):
        """
        逐笔行情回调 (可选)

        Args:
            tick: 逐笔行情数据
        """
        pass

    async def on_order(self, order: Order):
        """
        订单状态更新回调 (可选)

        Args:
            order: 订单对象
        """
        pass

    async def on_position(self, position: Position):
        """
        持仓变化回调 (可选)

        Args:
            position: 持仓对象
        """
        pass

    async def on_error(self, error: Exception):
        """
        错误回调 (可选)

        Args:
            error: 异常对象
        """
        logger.error(f"策略错误: {error}")

    async def on_stop(self):
        """
        策略停止回调
        在策略停止时调用一次,用于清理资源
        """
        logger.info("策略停止")


# ============ 示例策略 ============

class SimpleMovingAverageStrategy(BaseStrategy):
    """
    简单均线策略示例
    当短期均线上穿长期均线时买入,下穿时卖出
    """

    async def on_init(self):
        self.short_period = 5
        self.long_period = 20
        self.price_history: Dict[str, List[float]] = {}

        for symbol in self.config.symbols:
            self.price_history[symbol] = []

        logger.info(f"均线策略初始化完成, 短期={self.short_period}, 长期={self.long_period}")

    async def on_bar(self, bar: Bar):
        symbol = bar.symbol

        # 记录价格
        self.price_history[symbol].append(float(bar.close))

        # 保持历史数据长度
        if len(self.price_history[symbol]) > self.long_period:
            self.price_history[symbol].pop(0)

        # 计算均线
        if len(self.price_history[symbol]) >= self.long_period:
            short_ma = sum(self.price_history[symbol][-self.short_period:]) / self.short_period
            long_ma = sum(self.price_history[symbol][-self.long_period:]) / self.long_period

            # 获取当前持仓
            positions = await self.ctx.get_positions(symbol)
            current_position = positions[0] if positions else None

            # 交易逻辑
            if short_ma > long_ma and (not current_position or current_position.quantity <= 0):
                # 金叉买入
                await self.ctx.place_order(
                    symbol=symbol,
                    side=OrderSide.BUY,
                    order_type=OrderType.MARKET,
                    quantity=Decimal("0.001")  # 买入0.001 BTC
                )
                logger.info(f"买入信号: {symbol}, 短期MA={short_ma:.2f}, 长期MA={long_ma:.2f}")

            elif short_ma < long_ma and current_position and current_position.quantity > 0:
                # 死叉卖出
                await self.ctx.place_order(
                    symbol=symbol,
                    side=OrderSide.SELL,
                    order_type=OrderType.MARKET,
                    quantity=current_position.quantity
                )
                logger.info(f"卖出信号: {symbol}, 短期MA={short_ma:.2f}, 长期MA={long_ma:.2f}")

    async def on_stop(self):
        logger.info("均线策略停止, 清理资源")


# 策略入口函数 (系统调用)
def create_strategy(config: StrategyConfig, context: Context) -> BaseStrategy:
    """
    策略工厂函数

    Args:
        config: 策略配置
        context: 策略上下文

    Returns:
        策略实例
    """
    return SimpleMovingAverageStrategy(config, context)
```

## 5. 安全性设计

### 5.1 API Key 安全存储

```python
from cryptography.hazmat.primitives.ciphers.aead import AESGCM
from cryptography.hazmat.primitives.kdf.pbkdf2 import PBKDF2HMAC
from cryptography.hazmat.primitives import hashes
from cryptography.hazmat.backends import default_backend
import os
import base64


class APIKeyEncryptor:
    """API Key加密/解密器"""

    def __init__(self, master_key_env: str = "MASTER_KEY"):
        """
        初始化加密器

        Args:
            master_key_env: 存储主密钥的环境变量名
        """
        self.master_key = os.getenv(master_key_env)
        if not self.master_key:
            raise ValueError(f"环境变量 {master_key_env} 未设置")

        # 从Base64解码主密钥
        self.master_key = base64.b64decode(self.master_key.encode())

    def encrypt(self, plaintext: str) -> str:
        """
        加密明文

        Args:
            plaintext: 明文 (如API Key)

        Returns:
            Base64编码的密文
        """
        # AES-256-GCM加密
        aesgcm = AESGCM(self.master_key)
        nonce = os.urandom(12)  # 96位nonce
        ciphertext = aesgcm.encrypt(nonce, plaintext.encode(), None)

        # 拼接nonce和ciphertext,并Base64编码
        combined = nonce + ciphertext
        return base64.b64encode(combined).decode()

    def decrypt(self, ciphertext: str) -> str:
        """
        解密密文

        Args:
            ciphertext: Base64编码的密文

        Returns:
            明文
        """
        # Base64解码
        combined = base64.b64decode(ciphertext.encode())

        # 分离nonce和ciphertext
        nonce = combined[:12]
        ciphertext_bytes = combined[12:]

        # AES-256-GCM解密
        aesgcm = AESGCM(self.master_key)
        plaintext = aesgcm.decrypt(nonce, ciphertext_bytes, None)

        return plaintext.decode()
```

### 5.2 策略代码安全检查

```python
import ast
from typing import List, Set


class SecurityValidator:
    """策略代码安全验证器"""

    # 禁止导入的模块
    FORBIDDEN_MODULES = {
        'os',
        'subprocess',
        'shutil',
        'sys',
        'socket',
        'http',
        'urllib',
        'requests',  # 策略不应直接发HTTP请求
    }

    # 允许的模块白名单
    ALLOWED_MODULES = {
        'math',
        'statistics',
        'datetime',
        'decimal',
        'typing',
        'dataclasses',
        'pandas',  # 数据分析
        'numpy',   # 数值计算
        'squant',  # 策略SDK
    }

    def validate(self, code: str) -> tuple[bool, List[str]]:
        """
        验证策略代码安全性

        Args:
            code: 策略代码

        Returns:
            (是否通过, 错误消息列表)
        """
        errors = []

        try:
            tree = ast.parse(code)
        except SyntaxError as e:
            return False, [f"语法错误: {e}"]

        # 检查导入语句
        for node in ast.walk(tree):
            if isinstance(node, ast.Import):
                for alias in node.names:
                    module_name = alias.name.split('.')[0]
                    if module_name in self.FORBIDDEN_MODULES:
                        errors.append(f"禁止导入模块: {module_name}")
                    elif module_name not in self.ALLOWED_MODULES:
                        errors.append(f"未授权模块: {module_name}")

            elif isinstance(node, ast.ImportFrom):
                module_name = node.module.split('.')[0] if node.module else ''
                if module_name in self.FORBIDDEN_MODULES:
                    errors.append(f"禁止从模块导入: {module_name}")
                elif module_name not in self.ALLOWED_MODULES:
                    errors.append(f"未授权模块导入: {module_name}")

            # 检查函数调用 (防止直接调用危险函数)
            elif isinstance(node, ast.Call):
                if isinstance(node.func, ast.Name):
                    func_name = node.func.id
                    if func_name in ['exec', 'eval', 'open']:
                        errors.append(f"禁止调用函数: {func_name}")

        return len(errors) == 0, errors
```

**⚠️ 静态检查的局限性:**
- 无法发现动态导入 (`__import__(user_input)`)
- 无法检测代码注入 (`eval(f"{user_input}")`)
- 无法防止无限循环
- 无法检测递归深度攻击

### 5.2.2 分层安全策略 (运行时安全)

为弥补静态检查的不足,实现**多层安全策略**:

```python
import asyncio
import resource
from typing import List


class StrategyRuntimeSecurity:
    """策略运行时安全管理器"""

    def __init__(self, max_cpu_percent: float = 50.0, max_memory_mb: int = 512):
        """
        初始化安全管理器

        Args:
            max_cpu_percent: 最大CPU使用率(%)
            max_memory_mb: 最大内存使用量(MB)
        """
        self.max_cpu_percent = max_cpu_percent
        self.max_memory_mb = max_memory_mb
        self.timeout_seconds = 300  # 默认超时5分钟

    async def run_with_timeout(
        self,
        strategy_func,
        timeout_seconds: int = None
    ):
        """
        策略运行超时控制

        Args:
            strategy_func: 策略函数
            timeout_seconds: 超时时间(秒)

        Returns:
            策略执行结果

        Raises:
            asyncio.TimeoutError: 超时异常
        """
        timeout = timeout_seconds or self.timeout_seconds

        try:
            return await asyncio.wait_for(strategy_func(), timeout=timeout)
        except asyncio.TimeoutError:
            logger.error(f"策略运行超时 ({timeout}秒), 强制终止")
            raise

    def set_resource_limits(self):
        """
        设置资源限制 (防止资源耗尽)

        仅适用于Linux/MacOS系统
        """
        import sys
        import resource

        try:
            # 设置最大CPU时间 (单位: 秒)
            # 防止无限循环
            resource.setrlimit(
                resource.RLIMIT_CPU,
                (self.timeout_seconds, self.timeout_seconds * 1.1)
            )

            # 设置最大内存 (单位: 字节)
            max_memory_bytes = self.max_memory_mb * 1024 * 1024
            resource.setrlimit(
                resource.RLIMIT_AS,
                (max_memory_bytes, max_memory_bytes * 1.1)
            )

            logger.info(f"资源限制设置: CPU={self.timeout_seconds}s, Memory={self.max_memory_mb}MB")

        except (OSError, ValueError) as e:
            logger.warning(f"设置资源限制失败: {e}")

    def check_resource_usage(self) -> dict:
        """
        检查当前资源使用情况

        Returns:
            资源使用统计
        """
        import psutil

        process = psutil.Process()

        cpu_percent = process.cpu_percent(interval=1)
        memory_info = process.memory_info()

        return {
            "cpu_percent": cpu_percent,
            "memory_mb": memory_info.rss / 1024 / 1024,
            "memory_percent": (memory_info.rss / psutil.virtual_memory().total) * 100,
            "num_threads": process.num_threads(),
            "num_fds": process.num_fds() if hasattr(process, 'num_fds') else 0
        }

    def is_resource_exceeded(self) -> tuple[bool, str]:
        """
        检查资源是否超限

        Returns:
            (是否超限, 超限原因)
        """
        usage = self.check_resource_usage()

        if usage["cpu_percent"] > self.max_cpu_percent * 2:  # 允许2倍短期峰值
            return True, f"CPU使用率过高: {usage['cpu_percent']:.1f}%"

        if usage["memory_mb"] > self.max_memory_mb:
            return True, f"内存使用超限: {usage['memory_mb']:.1f}MB"

        if usage["num_threads"] > 100:
            return True, f"线程数过多: {usage['num_threads']}"

        return False, ""
```

```python
from fastapi import Depends, HTTPException, status
from fastapi.security import HTTPBearer, HTTPAuthorizationCredentials
from jose import JWTError, jwt
from datetime import datetime, timedelta


# JWT配置
SECRET_KEY = "your-secret-key"  # 从环境变量读取
ALGORITHM = "HS256"
ACCESS_TOKEN_EXPIRE_MINUTES = 60 * 24 * 7  # 7天

security = HTTPBearer()


def create_access_token(data: dict) -> str:
    """创建JWT访问令牌"""
    to_encode = data.copy()
    expire = datetime.utcnow() + timedelta(minutes=ACCESS_TOKEN_EXPIRE_MINUTES)
    to_encode.update({"exp": expire})
    encoded_jwt = jwt.encode(to_encode, SECRET_KEY, algorithm=ALGORITHM)
    return encoded_jwt


def verify_token(credentials: HTTPAuthorizationCredentials = Depends(security)) -> str:
    """验证JWT令牌并返回用户ID"""
    try:
        payload = jwt.decode(
            credentials.credentials,
            SECRET_KEY,
            algorithms=[ALGORITHM]
        )
        user_id: str = payload.get("sub")
        if user_id is None:
            raise HTTPException(
                status_code=status.HTTP_401_UNAUTHORIZED,
                detail="无效的认证凭据"
            )
        return user_id
    except JWTError:
        raise HTTPException(
            status_code=status.HTTP_401_UNAUTHORIZED,
            detail="无效的认证凭据"
        )

        raise HTTPException(
            status_code=status.HTTP_404_NOT_FOUND,

        )
```
