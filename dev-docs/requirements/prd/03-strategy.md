# PRD - 策略中心

## 2.3.1 策略模板

**功能描述**: 提供标准化的策略开发模板，用户通过实现模板接口来开发自定义策略。

**策略模板需要支持的生命周期方法**:

| 方法 | 说明 | 是否必须 |
|------|------|----------|
| `on_init` | 策略初始化，用于初始化数据和状态 | 否 |
| `on_start` | 策略启动，运行前的准备工作 | 否 |
| `on_stop` | 策略停止，清理资源、保存状态 | 否 |
| `on_tick` | Tick 数据回调，适用于高频策略 | 否 |
| `on_bar` | K 线数据回调，策略主要逻辑入口 | **是** |
| `on_order` | 订单状态更新回调 | 否 |
| `on_trade` | 成交回调 | 否 |
| `on_error` | 异常处理回调 | 否 |

**策略上下文需要提供的能力**:

| 类别 | 能力 | 说明 |
|------|------|------|
| **数据获取** | 获取历史 K 线 | 获取指定数量的历史 K 线数据 |
| | 获取当前 K 线 | 获取最新一根 K 线 |
| | 获取技术指标 | 计算并返回技术指标值 |
| **交易操作** | 买入 | 支持市价单和限价单 |
| | 卖出 | 支持市价单和限价单 |
| | 取消订单 | 取消指定订单 |
| | 取消所有订单 | 取消所有挂单 |
| | 平仓 | 平掉当前持仓 |
| **信息查询** | 账户信息 | 查询账户余额 |
| | 持仓信息 | 查询当前持仓 |
| | 策略参数 | 获取策略配置参数 |
| **日志通知** | 记录日志 | 记录策略运行日志 |
| | 发送通知 | 发送消息通知 |

**功能要求**:

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| STR-001 | 提供策略模板基类 | P0 |
| STR-002 | 提供策略上下文接口 | P0 |
| STR-003 | 提供示例策略（双均线策略） | P0 |
| STR-004 | 提供策略开发文档 | P1 |
| STR-005 | 支持策略参数可视化配置 | P1 |

## 2.3.2 策略上传与校验

**功能描述**: 支持通过 Web 界面上传策略文件，并自动进行校验。

**校验规则**:

| 校验项 | 描述 | 错误级别 |
|--------|------|----------|
| 文件格式 | 必须是 .py 文件 | Error |
| 语法检查 | Python 语法正确性 | Error |
| 类继承 | 必须继承策略模板基类 | Error |
| 必要方法 | 必须实现 on_bar 方法 | Error |
| 元信息 | 必须定义 name 和 version | Error |
| 安全检查 | 禁止导入危险模块（os.system, subprocess 等） | Error |
| 沙箱测试 | 在隔离环境中试运行 | Warning |

**功能要求**:

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| STR-010 | 支持拖拽上传 .py 文件 | P0 |
| STR-011 | 语法校验并返回详细错误信息 | P0 |
| STR-012 | 安全性检查（禁用危险模块） | P0 |
| STR-013 | 沙箱环境试运行校验 | P1 |
| STR-014 | 校验通过后自动入库 | P0 |
| STR-015 | 支持在线代码编辑器 | P2 |
