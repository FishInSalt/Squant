# PRD - 非功能需求

## 3.1 性能需求

| 编号 | 需求描述 | 指标 |
|------|----------|------|
| NFR-001 | 行情数据延迟 | < 500ms（从交易所到前端展示） |
| NFR-002 | 订单执行延迟 | < 1s（从策略下单到交易所确认） |
| NFR-003 | 页面加载时间 | < 3s（首屏加载） |
| NFR-004 | 并发策略数量 | 支持同时运行 10+ 策略 |
| NFR-005 | 回测性能 | 1年日线数据回测 < 10s |

## 3.2 可靠性需求

| 编号 | 需求描述 | 指标 |
|------|----------|------|
| NFR-010 | 系统可用性 | 99.9%（不含交易所故障） |
| NFR-011 | 数据持久化 | 所有订单、交易记录不丢失 |
| NFR-012 | 策略进程隔离 | 单个策略崩溃不影响其他策略 |
| NFR-013 | 自动恢复 | 系统重启后自动恢复运行中的策略 |

## 3.3 安全需求

| 编号 | 需求描述 | 说明 |
|------|----------|------|
| NFR-020 | API Key 加密 | 使用安全的加密算法存储 |
| NFR-021 | 策略沙箱 | 策略代码在隔离环境运行，禁止危险操作 |
| NFR-022 | 网络安全 | 支持仅本地/内网访问模式 |
| NFR-023 | 操作日志 | 关键操作记录审计日志 |

## 3.4 可维护性需求

| 编号 | 需求描述 | 说明 |
|------|----------|------|
| NFR-030 | 日志完善 | 结构化日志，便于问题排查 |
| NFR-031 | 配置外置 | 支持配置文件和环境变量 |
| NFR-032 | 容器化部署 | 支持 Docker 一键部署 |

## 3.5 可扩展性需求

| 编号 | 需求描述 | 说明 |
|------|----------|------|
| NFR-040 | 交易所扩展 | 架构支持方便地添加新交易所 |
| NFR-041 | 策略扩展 | 标准化策略接口，易于开发新策略 |
| NFR-042 | 通知渠道扩展 | 支持添加新的通知渠道 |
| NFR-043 | 市场扩展 | 架构支持后续添加港股、美股市场 |
