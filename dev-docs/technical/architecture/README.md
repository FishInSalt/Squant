## 核心设计原则

| 原则 | 说明 |
|------|------|
| **单体优先** | 个人项目不需要微服务，单体应用足够 |
| **进程隔离** | 策略运行在独立进程，崩溃不影响主服务 |
| **异步优先** | I/O 密集型操作使用 async/await |
| **关注点分离** | 清晰的分层架构，各层职责明确 |
| **可扩展** | 交易所、策略、通知渠道易于扩展 |
