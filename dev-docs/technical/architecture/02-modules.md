## 模块职责

| 模块 | 职责 | 关键类/函数 |
|------|------|-------------|
| **MarketService** | 行情数据获取、缓存、推送 | `get_ticker()`, `get_klines()`, `subscribe()` |
| **StrategyService** | 策略上传、校验、管理 | `upload()`, `validate()`, `list()` |
| **TradingService** | 回测/模拟/实盘运行管理 | `start_backtest()`, `start_live()`, `stop()` |
| **OrderService** | 订单管理、状态同步 | `create()`, `cancel()`, `sync_status()` |
| **RiskService** | 风控规则检查、熔断 | `check_order()`, `trigger_circuit_breaker()` |
| **AccountService** | 交易所配置、资产查询 | `add_exchange()`, `get_balance()` |
| **SystemService** | 数据管理、日志、配置 | `download_data()`, `get_logs()` |
| **StrategyEngine** | 策略进程管理、执行 | `spawn_process()`, `kill_process()` |

## 分层说明

### Presentation Layer (表现层)

负责与外部交互：
- **REST API**: HTTP 接口
- **WebSocket**: 实时数据推送
- **Scheduler**: 定时任务触发

### Service Layer (服务层)

核心业务逻辑：
- 业务规则校验
- 跨模块协调
- 事务管理

### Infrastructure Layer (基础设施层)

技术实现细节：
- 数据库访问
- 缓存操作
- 外部 API 调用
- 消息发送
